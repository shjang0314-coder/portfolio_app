# 1페이지 포트폴리오 웹사이트 기능 명세 (핵심)

## 1) 목적/기획 의도

* **목표**: 모던·전문 이미지 전달, 이력/프로젝트/연락 유도(CTA) 최적화
* **1페이지 구조**: Hero → About → Skills → Experience → Projects → Testimonials(선택) → Contact → Footer
* **KPI**: LCP ≤ 2.5s, CLS ≤ 0.1, 접근성/SEO 점수 90+ (Lighthouse 기준), 폼 전송 성공률 99%+

---

## 2) 기술 스택/환경

* **프론트엔드**: React + Vite(또는 Next.js SSG), Tailwind CSS, Framer Motion(애니메이션)
* **아이콘/폰트**: Lucide Icons, Variable font(예: Inter)
* **배포/CI**: GitHub Actions → Vercel/Netlify, 환경변수 관리(.env)
* **분석/로그**: Plausible/GA4(선택), Sentry(에러 로깅)
* **데이터 소스**: `/content/*.json` 또는 Markdown로 정적 관리

---

## 3) 공통 기능 요구사항

* **반응형**: 모바일(360px)～데스크톱(≥1440px) 브레이크포인트
* **다크모드**: `prefers-color-scheme` 자동 + 토글(로컬스토리지 기억)
* **접근성**: WCAG 2.1 AA, 키보드 내비게이션, ARIA 라벨, 명도 대비 4.5:1+
* **성능**: 이미지 최적화(WebP/AVIF, LQIP), 코드 스플리팅, 지연 로딩
* **SEO**: 메타/OG/Twitter 카드, sitemap.xml, robots.txt, canonical
* **구조화 데이터**: JSON-LD(개인/프로필, 프로젝트)
* **보안**: reCAPTCHA v3(또는 hCaptcha), 기본 보안 헤더, 폼 rate limit
* **애니메이션**: 섹션 페이드/슬라이드 인, 스크롤 트리거(감도 낮춤, 과도 X)
* **스크롤 UX**: 스무스 스크롤, 현재 섹션 네비 활성화(IntersectionObserver)

---

## 4) 레이아웃/내비게이션

* **헤더**: 로고/이름, 섹션 앵커 메뉴, 다크모드 토글, “Contact” 고정 CTA 버튼
* **고정 네비**: 상단 고정, 64px 높이, 스크롤 시 축소(Shadow 추가)
* **푸터**: 저작권, 소셜 링크, 이메일, Last updated 날짜

---

## 5) 섹션별 기능 명세

### 5.1 Hero

* **요소**: 이름/직함, 핵심 한 줄 소개(키워드 3개 하이라이트), 주요 CTA 2개(“이력서 다운로드”, “연락하기”)
* **배경**: 미세 그라디언트/노이즈 텍스처, 성능 저하 없는 벡터 데코
* **이력서**: `/public/resume.pdf` 제공(버전 관리 가능)

### 5.2 About

* **콘텐츠**: 짧은 자기소개(300자 내), 핵심 역량/도메인, 사진(라이트/다크 테두리)
* **하이라이트 배지**: 총 경력 연수, 주요 기술 스택, 관심 분야 태그

### 5.3 Skills

* **형식**: 카테고리(Frontend/Backend/DevOps/PM 등)별 태그 + 숙련도(아이콘 1~3 단계)
* **도구 팁**: 태그 hover 시 최근 사용 연도/프로젝트 링크 툴팁

### 5.4 Experience (경력/타임라인)

* **타임라인 카드**: 회사/역할/기간/성과(수치), 핵심 스택
* **접기/펼치기**: 세부 성과를 토글로 확장(기본 3줄)

### 5.5 Projects

* **그리드 카드(2~6개)**: 썸네일, 제목, 한줄 설명, 역할, 스택 배지
* **상호작용**: 카드 hover 모션, Detail 모달(문제–해결–성과, 링크, GitHub)
* **필터**: 카테고리/스택 필터(다중 선택)

### 5.6 Testimonials (선택)

* **슬라이더**: 추천인 사진/이름/직함/인용문(최대 200자), 자동/수동 넘김
* **신뢰 표식**: 로고(흑백), alt 텍스트 제공

### 5.7 Contact

* **폼 필드**: 이름, 이메일, 메시지(필수), 파일 업로드(선택, 5MB 제한)
* **유효성**: 즉시 검증 + 오류 메시지, 성공/실패 토스트
* **전송 방식**: 서버리스 함수(API) 또는 외부 서비스(EmailJS/Formspree)
* **추가 채널**: 이메일 클릭 시 `mailto:`, 소셜 아이콘(LinkedIn, GitHub 등)

---

## 6) 콘텐츠/에셋 관리

* **콘텐츠 스키마(JSON 예시)**

  * `profile.json`: 이름, 직함, 소개, 링크
  * `skills.json`: 카테고리, 태그, 레벨, 최근 사용
  * `experience.json`: 회사/역할/기간/성과(정량)
  * `projects.json`: 제목/설명/스택/링크/썸네일경로/핵심 포인트
  * `testimonials.json`: 인용/작성자/직함/아바타
* **이미지 규격**: 프로젝트 썸네일 1200×630, WebP/AVIF 기본, 1x/2x 준비

---

## 7) 폼/백엔드(경량)

* **엔드포인트**: `POST /api/contact`
* **검증**: 서버측 이메일 형식/본문 길이, 스팸 점수 체크
* **알림**: 성공 시 이메일 전달 + Slack/Webhook 알림(선택)
* **저장**: 필요 시 서버리스 KV/Sheet에 로그(PII 최소화, 30일 보관)

---

## 8) 성능/SEO/접근성 기준

* **성능**: 프리로드 핵심 폰트, 크리티컬 CSS 인라인, 비동기 스크립트
* **SEO**: 페이지 타이틀 규칙 `이름 | 직함 | 키워드`, OG 이미지 자동 생성
* **i18n(선택)**: ko/en 토글, 경로 기반(`/en`) 또는 쿠키

---

## 9) 품질 보증/테스트

* **브라우저 지원**: 최신 Chrome/Edge/Safari/Firefox, iOS 16+, Android 12+
* **E2E**: 주요 플로우(C TA 클릭, 폼 전송, 필터링) Cypress 테스트
* **접근성 테스트**: Axe/Lighthouse, 키보드 탭 순서 검수
* **시나리오**:

  * 폼 빈 값 전송 시 에러 노출
  * 오프라인 상태 이미지 캐시 동작(기본 PWA 옵션 선택)
  * 다크모드 전환 후 새로고침 시 상태 유지

---

## 10) 구성/디자인 가이드(핵심만)

* **컨테이너**: 최대 폭 1200px, 섹션 패딩 Y 96px
* **타이포**: H1 48–64, H2 28–36, 본문 16–18, 줄간 1.6
* **컬러**: 중성 베이스(그레이/네이비) + 포인트 1개(CTA), 다크모드 대비 강화
* **카드 스타일**: radius 16–20, soft shadow, 호버 엘리베이션
* **모션**: 200–300ms, 축적 애니메이션 지양(멀미 방지)

---

## 11) 운영/유지보수

* **버전/릴리즈**: Conventional Commits, 자동 배포 프리뷰 → 프로덕션
* **모니터링**: Uptime 체크(5분 주기), Sentry 에러 슬랙 알림
* **콘텐츠 업데이트**: `/content` 수정 → PR → 자동 배포

---

## 12) 전달물/산출물

* 소스 레포(Git), README(설치/빌드/배포 안내), 디자인 토큰, OG 템플릿
* 콘텐츠 샘플 JSON, 이미지 에셋(최적화본), 환경변수 예시(.env.sample)

## 13) 수락 기준(AC)

* Lighthouse 성능/접근성/SEO/Best Practices **각 90점 이상**
* 모바일/데스크톱 반응형 이상 없음
* 폼 전송: 정상/오류 케이스 처리 및 관리자 알림 수신 확인
* 프로젝트 필터/모달, 네비 섹션 하이라이트 정상 동작
* 다크모드 토글 유지(재방문 시 상태 기억)

> **비고**: 브랜드 컬러/로고, 실제 이력·프로젝트 데이터 확정 시 `/content`만 교체하여 재배포 가능합니다.
